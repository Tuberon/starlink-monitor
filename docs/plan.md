# План проекту

## Зроблено

- [x] Моніторинг dish: throughput, latency, obstruction, uptime, стан
      оновлення ПЗ, попередження, dish_id
- [x] Моніторинг router: окрема прошивка, стан оновлення, попередження
- [x] Автоматичний reboot (watchdog + update-ready для dish і router)
      з runtime-перемикачем і захистом від reboot-loop
- [x] Веб-дашборд: живі метрики, графіки історії, журнал подій
- [x] Системні метрики Pi (CPU/RAM/диск/температура)
- [x] Telegram: вихідні сповіщення (з підпис-фразами, що редагуються
      через веб-інтерфейс) + вхідні команди `/status`, `/reboot`, `/id`, `/help`
- [x] Одноразове сповіщення про підключення нової тарілки (dish_id)
- [x] Ручна перевірка стану оновлень з дашборду
- [x] Ручний reboot/shutdown самого Raspberry Pi з веб-інтерфейсу
- [x] Фізична кнопка виключення через GPIO (окремий сервіс, вимкнено за замовчуванням)
- [x] install.sh: install/update режими, детекція суттєвих змін,
      збереження signature_phrases.txt при оновленні
- [x] update.sh: ручне оновлення через sha256-перевірку архіву
- [x] Дедублікація однотипних подій у журналі (повтори того самого
      kind+message стискаються в один запис з лічильником)
- [x] Backup/restore налаштувань (Telegram config, auto-reboot,
      фрази підпису) одним JSON-файлом через веб-інтерфейс
- [x] Список під'єднаних WiFi-клієнтів роутера на дашборді
- [x] Запам'ятовування версій ПЗ dish/router по dish_id (`known_devices`) +
      команда `/id [ID]` в Telegram-боті (список пристроїв або деталі)
- [x] Окрема сторінка `/settings` (Telegram, фрази підпису, backup/restore)
- [x] Редагування параметрів `app/config.py` через веб-інтерфейс (`/settings`)
- [x] Перемикач додавання випадкових фраз підпису до Telegram-повідомлень
- [x] Фільтрація вибраних router alerts (не пишуться в БД/журнал/Telegram)
- [x] Опційне інтерактивне налаштування статичних IP (eth0/wlan0) в install.sh

## Можливі наступні кроки (не заплановані, лише ідеї)

- Розширена діагностика obstruction (мапа перешкод, не лише відсоток)
